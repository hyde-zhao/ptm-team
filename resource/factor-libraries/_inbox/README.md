# _inbox — 原始材料

## 用途

存放待处理的原始测试因子材料。典型来源：
- 手工整理的 Excel 测试因子表
- 从历史项目导出的 CSV 因子数据
- 需求文档中提取的测试变量草稿
- 缺陷分析中发现的遗漏因子记录

## 文件命名规则

```
{来源}_{日期}_{简要描述}.{ext}
```

示例：
- `tgfw_2026-06-03_策略路由因子表.xlsx`
- `project-x_2026-05-20_安全策略测试变量.csv`
- `defect_2026-04-15_遗漏因子记录.md`

## 处理流程

1. 将原始文件放入本目录
2. 对照 `../STANDARDS.md` 提取候选变量
3. 候选变量写入 `../_proposals/`
4. 处理完毕后，原始文件**不要删除**，保留作为追溯依据
5. 可选：在处理完成的文件名前添加 `DONE-` 前缀标记

## 注意

- 原始文件仅用于提取参考，**不直接作为因子库的一部分**
- 提取时注意去重：同一因子可能在多个 Excel 中以不同名称出现
- 提取结果写入 `_proposals/`，格式参考 `../_proposals/README.md`
