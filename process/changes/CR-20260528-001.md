# CR-20260528-001 - 场景发现方法论增强与人工决策显式化

## 变更请求摘要

用户提出三项需求，目标是在 ptm-tde 工作流的场景发现阶段增强方法论基础，并在人工确认门禁处显式化备选方案与决策收集要求。

### 需求 1：场景发现阶段引入 Superpowers Brainstorming 和 GSD Discuss-Phase 方法论

当前 `skills/scenario-discovery/SKILL.md` 的步骤 2（场景再发现与头脑风暴）已经包含 12 个维度的头脑风暴表格，但缺少以下外部方法论的显式指导：

**A. Superpowers Brainstorming 方式**（源自 `github.com/obra/superpowers`）：
- 先提出澄清问题，再展开设计
- 探索替代方案，而非直接跳到单一方案
- 以可消化的小块呈现设计结果
- 保存设计文档用于后续追溯

**B. GSD Discuss-Phase 方式**（源自 GSD 项目）：
- 通过结构化讨论捕获实现决策
- 将大任务拆分为独立执行的小上下文
- 避免 "context rot"（上下文退化）
- 每个子任务在干净的上下文窗口中运行

### 需求 2：人工确认时必须提供备选方案

Meta Flow 规则第 32 条已规定"每个决策项必须包含推荐方案、至少 1 个备选方案（优先 2 个）"，本需求要求在 ptm-tde 工作流中更明确地强制执行此规则，特别是在场景发现 Skill 的 `confirmation_gaps` 输出中。

### 需求 3：人工门禁触发时收集所有待确认问题

Meta Flow 规则第 32 条已规定"工作流程中所有需要人工确定的信息都必须形成决策项"，本需求要求在 ptm-tde 工作流中显式化此要求，确保人工门禁时不遗漏任何待确认问题。
