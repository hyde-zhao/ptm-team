## Change Request

用户上传 `.input/factor-library-design.md`，要求建设公共因子库，并调整 `ptm-tde` agent、流程和 Skill 消费方式。用户确认：

- 因子库归档到 `ptm-team` 项目。
- 因子库直接挂在仓库级 `resource/` 目录，不放在 `ptm-tde` 专属目录。
- `ptm-tde` 与因子库建立显式关联关系。
- 安装 `ptm-tde` agent 时同步安装关联因子库，卸载时按引用关系卸载。
- 因子库未来不只给 `ptm-tde` 使用。

## Review Feedback - Topology Role / Factor Separation

用户在审查公共因子库消费链路后提出整改：MFQ 链路必须区分测试因子、拓扑角色和真实组网对象，避免把 `DUT.port1`、`DUT.port2`、`TG.port1` 等 TOPO 实例误写成因子取值。

本轮硬约束：

> CAE 负责表达测试逻辑和拓扑角色；测试整合负责从 `analysis/scenarios/confirmed-scenarios.md` 绑定真实组网；PPDCS/PC 负责消费绑定并生成可执行步骤。任何真实端口引用必须能回链到 `confirmed-scenarios.md`，否则只能是 `needs-confirmation`。
