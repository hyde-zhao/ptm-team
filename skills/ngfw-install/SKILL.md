# NGFW 卸载安装工具

## 概述

用户会要求你对 NGFW (或 TGWF) 进行卸载安装，因为卸载安装整个过程都是相对固化的，只需要使用 `scripts/ngfwinstall_tool.py` 这个工具就可以完成整个步骤，所以你只需要确保执行环境能够正常使用 uv，同时在调用工具时候的入参使用正确即可。

## 重要提示

**工具的存在是为了作为黑盒脚本被直接调用，而不是被读取到你的上下文中**

## 工作流

### 准备

检查当前环境是否安装了 uv，可以使用 `uv --version` 命令来检查，如果当前环境没有安装 uv 的话，则需要提示用户查阅 uv 的[官方文档](https://github.com/astral-sh/uv)来安装 uv

### 查看工具帮助信息

当你在使用这个脚本之前，务必要先使用 uv 运行 `scripts/ngfwinstall_tool.py --help` 了解一下有关这个脚本的使用帮助。

### 确认调用入参

用户在调用这个工具的时候，必须要提供以下入参：

- `serial-url`: 设备使用的串口地址
- `package-url`: 安装包的 URL
- `device-type`: 设备的型号
- `device-ip`: 设备的管理IP
- `md5`: 安装包的 MD5 校验值（可选）

如果有其他必选的入参，就参考 `scripts/ngfwinstall_tool.py --help` 的输出来补充

如果用户在对话的过程中带上了完整的入参信息，那么可以直接进入下一个步骤。如果没有提供完整的入参信息，则需要向用户请求补充缺失的入参信息。

### 开始执行

在开始执行之前，除非存在以下情况，你可以不需要用户二次同意，直接开始执行脚本：

- 用户没有明确的告知直接执行
- 没有在首次对话中提供所有完整的信息
- 使用代词指定操作自动化工厂的设备，而且代词和工厂设备目标名称完全匹配

其他情况下你都需要完整的输出所有入参信息，让用户二次同意之后，才可以开始执行。

运行脚本之前，不需要切换工作路径，你直接在当前项目的工作路径下运行脚本即可，不要在 skill 的目录中运行

同时因为这个脚本运行的时间是比较长的（600s 以上），你应该在一个单独的 shell 中运行这个脚本，同时在执行的过程中实时监听它输出的内容，根据输出的内容来判断当前执行的状态，并及时给用户同步。

如果执行失败了，你可以初步的看一下脚本输出的内容，判断失败的原因并告知用户，同时这个脚本还会把完整的安装日志记录到 `logs/` 目录中。
日志文件名由串口服务器地址（`serial-url`）自动生成URL 中的特殊字符（`/`、`:` 等）均替换为 `_`，例如 `telnet://<IP_ADDRESS>:10008` 对应的日志文件为 `logs/telnet_<IP_ADDRESS>_10008.log`。可以引导用户查看对应的日志文件来排查问题。
