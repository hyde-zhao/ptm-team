## 目标

对设计计划中 PPDCS 特征为 **D-Data** 的逻辑用例，执行五步设计过程，
等价类划分→等价类隔离+边界值识别（三点法）→LC→三类数据分配→输出物理用例。

## 理论基础

D-Data 是 PPDCS 五特征之一：
> 被测功能的输入数据有明确的取值范围，各数据项相对独立，
> 每个数据项可单独验证有效性。

**识别条件**：需求描述中包含"取值范围"、"有效值"、"最大最小"等词，
且各参数之间没有业务规则依赖。

**关键区分**：
- D-Data vs P-Parameter — 参数间有规则？有 = Parameter，无/独立 = Data
- D-Data vs C-Combination — 独立验证够？够 = Data，需交互组合 = Combination

**建模工具**：等价类划分表 + 边界值分析

## 五步用例设计过程

读取 `test-point-integrator` 输出的逻辑用例（含因子-取值表和动作路径），执行以下五步：

### 第一步：测试数据（因子-取值表补全 + 等价类细化）

从整合阶段的因子-取值表出发，补充因子类型，并细化等价类（有效/边界/无效）：

```markdown
| 因子 | 取值 | 类型 | 等价类 |
|------|------|------|--------|
| 日志保存天数 | 1 | 参数 | 有效（最小边界） |
| 日志保存天数 | 2 | 参数 | 有效（最小+1） |
| 日志保存天数 | 30 | 参数 | 有效（典型值） |
| 日志保存天数 | 364 | 参数 | 有效（最大-1） |
| 日志保存天数 | 365 | 参数 | 有效（最大边界） |
| 日志保存天数 | 0 | 参数 | 无效（最小-1） |
| 日志保存天数 | 366 | 参数 | 无效（最大+1） |
| 日志保存天数 | -1 | 参数 | 无效（负数） |
| 日志保存天数 | 1.5 | 参数 | 无效（非整数） |
```

**边界值选取规则**：`min, min+1, 典型值, max-1, max`（有效区5点）+ `min-1, max+1`（无效边界2点）+ 特殊值（0, 空, null, 负数, 浮点数）

### 第二步：等价类隔离+边界值识别

**D-Data vs C-Combination 决策提示（先判断，再设计）**：

```
判断当前数据项是否需要 Combination：
- 数据项A 和 数据项B 是否存在交叉约束（A=x 时 B 必须=y）？
  → 是：该参数对应切换至 combination-design
  → 否：继续使用 data-design（各项独立）
- 数据项A 的有效性是否取决于其他参数的值？
  → 是：切换至 parameter-design（存在规则依赖）
  → 否：继续 data-design
```

#### 2.1 等价类隔离

对每个数据项进行等价类划分，**有效等价类可合并测试，无效等价类必须隔离（每次只设一个无效值）**：

| 等价类类型 | 含义 | 测试原则 |
|-----------|------|---------|
| 有效等价类（EVP） | 取值范围内的合法值 | 可合并至同一用例 |
| 无效等价类（IVP） | 取值范围外的非法值 | 必须隔离，一次一个 |
| 边界等价类（BVP） | 范围的边界取值 | 优先测试，细分为三点 |

#### 2.2 边界值三点法（上点/离点/内点）

对每个有数值范围的数据项，使用**三点法**精确识别边界用例：

| 点类型 | 定义 | 对应有效范围 [min, max] |
|--------|------|----------------------|
| **上点（On Point）** | 边界值本身 | min 和 max |
| **离点（Off Point）** | 超出边界1步 | min-1 和 max+1 |
| **内点（In Point）** | 范围内典型值 | min+1 和 max-1 |

**示例（日志保存天数，有效范围 [1, 365]）**：

| 点类型 | 取值 | 类型 | 等价类 |
|--------|------|------|--------|
| 上点（下边界） | 1 | 参数 | 边界有效 |
| 内点（下边界） | 2 | 参数 | 有效 |
| 内点（典型值） | 30 | 参数 | 有效 |
| 内点（上边界） | 364 | 参数 | 有效 |
| 上点（上边界） | 365 | 参数 | 边界有效 |
| 离点（下超界） | 0 | 参数 | **无效（隔离）** |
| 离点（上超界） | 366 | 参数 | **无效（隔离）** |
| 特殊无效值 | -1 | 参数 | 无效（负数，隔离） |
| 特殊无效值 | 1.5 | 参数 | 无效（非整数，隔离） |
| 特殊无效值 | 空 | 参数 | 无效（空值，隔离） |

**D-Data 的操作动作独立于数据取值**，通常为 1 条路径：

```
路径 P1（唯一路径）：
1. 进入日志配置 → 日志保存设置页面
2. 将目标参数修改为测试数据值
3. 点击"保存"
4. 刷新页面，检查保存结果
```

若多个数据项的配置页面不同，可枚举多条路径（每个数据项一条），但同一路径不做组合。

### 第三步：数据组合分析（D-Data 等价类策略）

**D-Data 的组合策略**：各数据项**独立验证**，不做跨数据项组合（与 C-Combination 的核心区别）

| 策略类型 | 适用条件 | 说明 |
|---------|---------|------|
| **有效值合并** | 有效等价类 | 多个有效数据项可在同一条用例中配置 |
| **无效值隔离** | 无效等价类 | 每次只测一个无效值，其余为有效值 |
| **边界值优先** | 等价类边界 | 优先测试 min, max, min-1, max+1 |

**组合约束**（D-Data 通常无交叉约束，但若有则记录）：
```
无约束（各数据项完全独立）
```

**全量组合结果**（等价类驱动，非全排列）：

| 组合编号 | 日志保存天数 | 日志文件大小 | 等价类说明 | 预期结果 |
|---------|------------|------------|----------|---------|
| C-01 | 30 | 512 | 全有效（典型值组合） | 配置成功 |
| C-02 | 1 | 512 | 保存天数最小边界 | 配置成功 |
| C-03 | 365 | 512 | 保存天数最大边界 | 配置成功 |
| C-04 | 0 | 512 | 保存天数=0（无效，隔离） | 提示输入无效 |
| C-05 | 366 | 512 | 保存天数超上限（无效，隔离） | 提示超出范围 |
| C-06 | -1 | 512 | 保存天数负数（无效，隔离） | 提示输入无效 |
| C-07 | 空 | 512 | 保存天数为空（无效，隔离） | 提示不能为空 |
| C-08 | 30 | 1 | 文件大小最小边界 | 配置成功 |
| C-09 | 30 | 0 | 文件大小=0（无效，隔离） | 提示输入无效 |

### 第四步：三类数据分配

**D-Data 用例按三类数据组织输出**：将等价类和边界值识别的结果，分三类分配到具体用例中：

| 数据类型 | 含义 | 用例数量策略 |
|---------|------|-----------|
| **有效等价类（典型值）** | 范围内的代表性合法值 | 合并测试：多个有效参数共一条用例 |
| **无效等价类** | 范围外的非法值、类型错误等 | 隔离测试：每类无效值单独一条用例 |
| **边界值（三点法）** | 上点（边界值本身）、离点（超出1步）、内点（范围内1步） | 上点和离点优先；内点可合并进典型值用例 |

**三类数据分配表**（以日志保存天数+文件大小为例）：

| 数据类型 | 保存天数 | 文件大小 | 等价类说明 | 策略 |
|---------|---------|---------|----------|------|
| 有效典型值 | 30 | 512 | 全有效合并 | 一条用例 |
| 边界值-上点（下） | 1 | 512 | 保存天数min上点 | 一条用例 |
| 边界值-上点（上） | 365 | 512 | 保存天数max上点 | 一条用例 |
| 边界值-内点（下） | 2 | 512 | 保存天数min+1 | 可并入典型值 |
| 无效-离点（下） | 0 | 512 | 保存天数min-1离点（隔离） | 一条用例 |
| 无效-离点（上） | 366 | 512 | 保存天数max+1离点（隔离） | 一条用例 |
| 无效-类型错误 | -1 | 512 | 负数（隔离） | 一条用例 |
| 无效-空值 | 空 | 512 | 空值（隔离） | 一条用例 |
| 有效-另项边界 | 30 | 1 | 文件大小min上点 | 一条用例 |
| 无效-另项离点 | 30 | 0 | 文件大小min-1离点（隔离） | 一条用例 |

**D-Data 覆盖策略**：BA组合（有效值合并，无效值各自独立）

```
最终用例集：
- 有效等价类用例：典型值合并（C-01）+ 下边界上点（C-02）+ 上边界上点（C-03）+ 另项边界（C-08）
- 无效等价类用例：各无效类各一条（C-04离点/C-05超上限/C-06负数/C-07空值/C-09另项离点）
决策理由：D-Data 独立验证，无效值必须隔离，有效值可合并；BA组合平衡效率
```

## 优先级分配规则

| 用例类型 | 优先级 |

| 有效典型值 | P1 |
| 有效边界值（min, max） | P2 |

| 极端值 | P3~P4 |

## D-Data vs C-Combination 的设计差异

| 维度 | D-Data | C-Combination |
|------|--------|---------------|
| 因子关系 | 独立 | 需交互 |
| 验证方式 | 每个数据项单独验证 | 多因子组合验证 |
| 无效值测试 | 一次一个无效值 | 按组合策略 |
| 压缩需求 | 无（线性增长） | 需 Pairwise/正交压缩 |

## Gotchas

- D-Data 的核心是"独立性"：各数据项可单独验证，不需要组合
- 无效等价类必须隔离测试：一次只测一个无效值
- 如果发现某些数据项之间有规则依赖 → 应回退到 P-Parameter
- 如果发现需要组合测试 → 应回退到 C-Combination
- 边界值分析的"5+2"法则：min, min+1, typical, max-1, max（有效区）+ min-1, max+1（无效区）
